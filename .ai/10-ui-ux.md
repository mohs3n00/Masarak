# UI / UX Rules

The interface should be consistent, intuitive, and accessible.

Never sacrifice usability for visual appearance.

---

# General Principles

Every screen should feel like part of the same product.

Maintain consistency across the application.

---

# Design Consistency

Maintain:

Typography

Spacing

Colors

Border radius

Icons

Animations

Shadows

Layouts

Never redesign existing patterns without approval.

---

# User Experience

Every interaction should provide feedback.

Never leave users wondering whether something happened.

---

# Loading States

Every asynchronous action should provide visible loading feedback.

Buttons should indicate loading.

Pages should show loading placeholders or skeletons.

---

# Empty States

Never leave empty pages.

Explain why there is no data.

Suggest the next action.

---

# Error States

Display friendly messages.

Help users recover.

Never expose technical errors.

---

# Success States

Confirm successful actions.

Avoid excessive notifications.

---

# Buttons

Buttons must:

Have clear labels.

Be visually distinguishable.

Provide loading feedback.

Prevent duplicate clicks.

Be disabled only when appropriate.

---

# Forms

Forms must:

Validate immediately when appropriate.

Explain validation errors.

Preserve entered data after validation errors.

Prevent accidental data loss.

---

# Navigation

Navigation must be predictable.

Users should always know where they are.

Support browser back navigation.

Avoid dead ends.

---

# Tables

Support:

Pagination

Searching

Sorting

Filtering

Loading

Empty state

Error state

---

# Mobile Experience

Every page must be usable on:

Desktop

Laptop

Tablet

Mobile

Avoid horizontal scrolling.

---

# Accessibility

Use semantic HTML.

Support keyboard navigation.

Provide labels.

Provide alt text.

Maintain sufficient contrast.

Visible focus states are required.

---

# Colors

Never use color as the only indicator.

Support dark mode if the project does.

Maintain contrast ratios.

---

# Typography

Use consistent font sizes.

Maintain readable spacing.

Avoid tiny text.

---

# Animations

Animations should enhance usability.

Never slow users down.

Respect reduced motion preferences when applicable.

---

# Icons

Icons should always have meaning.

Avoid decorative icons without purpose.

---

# Modals

Trap focus.

Support Escape.

Restore focus after closing.

Prevent accidental dismissal of destructive actions.

---

# Confirmation

Require confirmation before:

Deleting

Resetting

Irreversible actions

---

# Notifications

Keep notifications:

Short

Useful

Actionable

Avoid notification spam.

---

# UX Checklist

✓ Consistent design

✓ Responsive layout

✓ Loading states

✓ Empty states

✓ Error states

✓ Success states

✓ Accessible

✓ Keyboard friendly

✓ No broken navigation

✓ No confusing interactions
